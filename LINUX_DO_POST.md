# 开源项目分享：法律 AI 助手 - 让判决书人人都能看懂 🤖⚖️

大家好！今天给大家分享一个我刚完成的开源项目：**法律 AI 助手**。

## 💡 项目背景

你有没有遇到过这种情况：

- 看法律判决书，满篇专业术语，完全看不懂？
- 想了解某个案件的判决逻辑，但法律文书太晦涩？
- 想知道"到底谁赢了"、"要赔多少钱"，但要翻半天才能找到？

作为一个普通人，我也有同样的困扰。所以我做了这个项目，用 AI 把法律判决书"翻译"成人话。

## 🎯 核心功能：双视角解读

这是项目最大的亮点！**同一个案件，提供两种完全不同的解读方式：**

### 👔 专业视角
给律师和法律专业人士看的：
```
本案系离婚纠纷案件。原告郑十以夫妻感情破裂为由诉请离婚，
被告钱十一不同意离婚。法院查明双方于2015年登记结婚，
婚后育有一子，因性格不合经常争吵...
```

### 👤 普通人视角
用大白话解释：
```
这是一个离婚官司。郑十（妻子）想跟钱十一（丈夫）离婚，
说两人性格不合老吵架，感情没了。丈夫不想离，说还能过下去。
但法院发现他们已经分居一年多了，去年起诉过一次没判离，
这次又来了，还是没和好。法院觉得这婚姻确实过不下去了，就判离了...
```

**法律条文也有通俗解释：**
```
《中华人民共和国民法典》第一千零八十七条：
离婚时，夫妻的共同财产由双方协议处理；协议不成的，
由人民法院根据财产的具体情况，按照照顾子女、女方和
无过错方权益的原则判决。

这条说的是财产怎么分：
夫妻离婚时，婚后挣的钱、买的东西原则上要平分。
但法院也会考虑实际情况，比如谁带孩子、谁更需要照顾等，
尽量公平合理地分配。
```

## 🤖 不只是提示词，是真正的 Agent

很多 AI 应用只是简单地调用 API，但这个项目实现了一个**真正的 AI Agent**：

```
🤖 Legal Analysis Agent 启动...
📊 Step 1: 提取案例关键要素...
🔍 Step 2: 搜索相似案例...
📚 Step 3: 查找相关法律依据...
🧠 Step 4: 综合分析判决...
✅ 分析完成！
```

Agent 会：
1. **自主决策**：根据案件内容决定分析步骤
2. **使用工具**：调用数据库搜索、法律查询等工具
3. **多步推理**：综合多个信息源进行分析
4. **状态管理**：记住之前的分析结果

## 🛠️ 技术栈

### 后端
- **FastAPI** - 现代化的 Python Web 框架
- **PostgreSQL** - 数据持久化
- **SQLAlchemy (异步)** - ORM
- **Claude API** - AI 分析引擎
- **Docker** - 容器化部署

### 前端
- **React 18 + TypeScript** - 类型安全的 UI
- **Ant Design** - 企业级 UI 组件
- **Zustand** - 轻量级状态管理
- **React Markdown** - Markdown 渲染

### 亮点
- ✅ 全异步架构，性能优秀
- ✅ Docker 一键部署
- ✅ 完整的类型定义
- ✅ RESTful API 设计

## 📚 数据来源

项目包含两类案例：

1. **真实案例**（绿色标签）
   - 来自最高人民法院公报
   - 指导性案例
   - 明确标注数据来源

2. **教学示例**（橙色标签）
   - 用于功能演示
   - 明确标注"仅供演示"

## 🚀 快速开始

```bash
# 1. 克隆项目
git clone https://github.com/yourusername/legal-ai-assistant.git
cd legal-ai-assistant

# 2. 配置 API Key
cp .env.example .env
# 编辑 .env，填入你的 ANTHROPIC_API_KEY

# 3. 一键启动
docker-compose up -d

# 4. 导入数据
docker-compose exec backend python scripts/import_sample_cases.py
docker-compose exec backend python scripts/add_real_cases.py

# 5. 访问应用
# 前端: http://localhost:5173
# 后端: http://localhost:8000/docs
```

默认账号：`testuser` / `123456`
